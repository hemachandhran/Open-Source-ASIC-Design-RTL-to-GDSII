# Caravel Test Block Verification

## Introduction

Caravel tests are used to verify the functionality of management SoC peripherals and integration features available within the Caravel environment. Each test executes dedicated firmware on the embedded VexRiscv processor and validates the corresponding hardware block using a dedicated testbench.

These tests help ensure correct operation of interfaces, memory subsystems, GPIO control, communication peripherals, and SoC-level functionality before moving to larger integration and tapeout flows.

## Conclusion

The tests-caravel verification environment was used to validate multiple management SoC peripherals and integration features. GPIO Management, HKSPI, HKSPI Power, Memory, Pass Through Fix, Pullupdown, SPI Master, SRAM Execute, UART, and User Pass Through tests completed successfully and produced the expected results. Overall, the majority of Caravel test blocks passed successfully, providing confidence in the functionality of the integrated management SoC peripherals and interfaces.
